# Kong Gateway Migration Plan

## Architecture Summary

| Current (Express)                                       | Target (Kong OSS DB-less)                                                                                                                                    |
| ------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `api-gateway/src/index.js` (228 lines, single-file)     | `services/kong/compose.yaml` + `config/kong.yml` (isolated compose + declarative YAML)                                                                       |
| `http-proxy-middleware` routing                         | Kong `services` + `routes` entities                                                                                                                          |
| `api-gateway` + `auth-service` in single compose        | `auth-service` managed via `services/auth-service/compose.yaml`, Kong managed via `services/kong/compose.yaml`. All connect to shared `app-network` external |
| `jsonwebtoken` middleware (with `/api/v1/*` bypass bug) | JWT validation pushed to each service (auth-service already has it; content-service needs it)                                                                |
| `express-rate-limit` + `rate-limit-redis`               | Kong `rate-limiting` plugin (Redis-backed)                                                                                                                   |
| `helmet` + `cors` packages                              | Kong `cors` plugin                                                                                                                                           |
| `console.log` request logging                           | Kong `file-log` or `http-log` plugin                                                                                                                         |
| No metrics                                              | Kong `prometheus` plugin → Prometheus → Grafana                                                                                                              |
| Stub services → 502                                     | Kong `request-termination` plugin → 503 JSON                                                                                                                 |

**Port mapping**: Keep external port `8081` → Kong proxy internal port `8000` (was `8080` on Express).

## Phase 3: Traffic Control & Load Balancing

| #   | Task                                              | Files                           | Description                                                                                                                                                                                                                                                                                                                             |
| --- | ------------------------------------------------- | ------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 3.1 | Configure `rate-limiting` plugin (global default) | `services/kong/config/kong.yml` | Global `rate-limiting` plugin: `policy: redis`, `redis_host: kong-redis`, `redis_port: 6379`, `fault_tolerant: true`, `hour: 200` (≈ current 200/15min). 2nd tier per-service overrides where needed. Note: `kong-redis` (container `cambridge-kong-redis`) replaces old `redis` to avoid name clash with auth-service's internal Redis |
| 3.2 | Configure per-route rate limit overrides          | `services/kong/config/kong.yml` | Auth routes get stricter (e.g., `minute: 10` on login to prevent brute-force). Content routes get permissive (e.g., `hour: 1000`). Use rate-limiting plugin on specific service/route level                                                                                                                                             |
| 3.3 | Set up active health checks                       | `services/kong/config/kong.yml` | For each upstream service: `healthchecks.active.type: http`, `healthchecks.active.http_path: /health`, `interval: 30`, `healthy.threshold: 1`, `unhealthy.threshold: 3`. Unhealthy targets auto-removed from load balancing                                                                                                             |
| 3.4 | Configure load balancing with `upstreams`         | `services/kong/config/kong.yml` | Wrap critical services (auth, content) in `upstreams` with `algorithm: round-robin` or `least-connections`. Add `targets` pointing to `host:port`. Prepares for horizontal scaling later                                                                                                                                                |
| 3.5 | Configure request/response size limits            | `services/kong/config/kong.yml` | Global `request-size-limiting` plugin: `allowed_payload_size: 10` (MB). Prevents oversized requests                                                                                                                                                                                                                                     |
| 3.6 | Configure timeouts                                | `services/kong/config/kong.yml` | Per-service: `connect_timeout: 5000`, `read_timeout: 60000`, `write_timeout: 60000`, `retries: 3`. Matches current gateway proxy timeout config                                                                                                                                                                                         |
| 3.7 | **Verify**: Rate limiting and health checks       | Stress test                     | Send 201 requests in 1 hour → 200th succeeds, 201st gets 429. Temporarily stop auth-service → health check detects it → routes fail fast                                                                                                                                                                                                |

| 4.4 | Add Prometheus plugin to Kong                           | `services/kong/config/kong.yml`           | Global `prometheus` plugin: no special config needed. | 4.5 | Add Prometheus + Grafana to Kong compose                | `services/kong/compose.yaml`              | Add `prom/prometheus` (scraping `kong:8001/metrics`, port `9090`) + `grafana/grafana` (Prometheus datasource, Kong dashboard 7424, port `3000`). Replaces current `console.log("[Gateway] ...")`                                                                                                                |

Generate traffic → metrics populate                                                                                                                                              |
